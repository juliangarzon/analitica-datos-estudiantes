What Are Large Language Models?

LLMs are trained on massive amounts of text. They predict the next token (word/piece) based on everything that came before. Think of it as autocomplete on steroids.

Good At

  • Generating code from descriptions
  • Writing and formatting documentation
  • Translating between formats (Markdown to HTML, JSON to CSV)
  • Explaining concepts and debugging errors
  • Following structured templates consistently

Bad At

  • Math and precise calculations
  • Real-time or live data (no internet access)
  • Guaranteeing 100% accuracy
  • Understanding YOUR specific project (without context)
  • Knowing what changed since their training cutoff

The Context Window

Your conversation is the AI's working memory. Bigger context window = more of your project the AI can see at once. This is why tool choice matters.

One-off question. No memory. No files.

What Is an Agent?

An agent is an AI that can take actions, not just generate text. Think of it as a junior developer who follows instructions precisely but needs clear direction.

What an Agent Can Do

  • Read files in your project
  • Write and edit code and documents
  • Run commands (scripts, tests, installs)
  • Make decisions based on what it finds
  • Execute multi-step plans autonomously

Examples in Practice

  • A data analyst agent that generates EDA reports
  • A documentation agent that writes READMEs from code
  • A testing agent that writes and runs test suites
  • A code reviewer agent that checks quality and patterns

Key insight: agents need clear instructions, just like any team member. Vague instructions produce vague results.

What Is a Skill?

A skill is a SKILL.md file -- a markdown document with instructions, constraints, and examples that tells an agent how to perform a specific task. Think of it as a recipe card for AI.

The Agent Skills Open Standard

The agentskills.io standard defines a universal format. One SKILL.md file works across 20+ tools:

OpenCode

Claude Code

Gemini CLI

Codex CLI

What a SKILL.md Contains

  • Name and description of the task
  • Instructions the agent must follow
  • Constraints and boundaries
  • Examples of expected output

Why Skills Matter

  • Consistency: same quality every time
  • Shareable: your team uses the same skill
  • Portable: switch AI tools, keep skills
  • Versionable: stored in your repo with git

How to Create a Skill (Step by Step)

Step 1: Create the folder

.opencode/skills/my-skill/

Step 2: Create SKILL.md with frontmatter

Name, description, version (YAML between --- markers)

Step 3: Write instructions

Be specific! Tell the AI exactly what to do.

Step 4: Define the output format

Show the exact structure you want back.

Step 5: Add rules and constraints

What the AI must NOT do. Boundaries matter.

Step 6: Test and iterate

Run it, see the output, improve, repeat.

--- name: Data Quality Report description: Generates a data quality report for a dataset version: 1.0.0 --- # Data Quality Report Analyze the provided dataset and generate a comprehensive data quality report. ## Format ### Dataset Overview - Name: [dataset name] - Rows: [count] - Columns: [count] ### Column Analysis For each column: - Type: [detected type] - Missing: [count] ([percentage]%) - Unique: [count] - Issues: [list any problems] ### Quality Score - Completeness: [X/10] - Consistency: [X/10] - Validity: [X/10] - Overall: [X/10] ### Recommended Actions 1. [First action] 2. [Second action] ## Rules - Always check for missing values, duplicates, and type mismatches - Score must be justified with evidence - Recommend concrete actions, not vague suggestions

The more specific your instructions and format, the better the output. First version is never the best -- iterate.

Skills Work Across Domains

Skills are not just for data analytics. The same format works for any repeatable task.

Data Analytics Skill

--- name: EDA Summary Generator description: Generates an EDA summary from a dataset version: 1.0.0 --- # EDA Summary Analyze the dataset and produce a structured EDA report. ## Format ### Distribution Analysis ### Correlations Found ### Outliers Detected ### Key Insights (top 5) ### Suggested Next Steps ## Rules - Use actual column names - Include specific numbers - Prioritize actionable insights - Flag data quality issues first

Software Development Skill

--- name: README Generator description: Creates a project README from codebase analysis version: 1.0.0 --- # README Generator Analyze the project and generate a README.md. ## Format # [Project Name] [One-line description] ## Installation [Steps] ## Usage [Examples] ## Contributing [Guidelines] ## Rules - Read package.json or setup.py for project metadata - Include actual commands, not placeholders - Keep it under 200 lines

Same Structure, Different Domains

Every skill has the same anatomy: frontmatter (metadata), instructions (what to do), format (output structure), and rules (constraints). Master the pattern once, apply it everywhere.

What Is an MCP Server?

MCP (Model Context Protocol) lets your AI agent connect to external tools and data sources. Instead of copy-pasting data into prompts, you give the agent a live connection. Think of it as giving your agent hands to interact with the outside world.

What MCP Servers Provide

  • Live access to databases, APIs, and files
  • Run commands in external tools (GitHub, Jira, Slack)
  • Query real-time data instead of stale context
  • Bridge between your agent and any service with an API

Examples

  • A GitHub MCP server: agent reads issues, creates PRs
  • A database MCP server: agent queries your schema live
  • A filesystem MCP server: agent navigates your codebase
  • A browser MCP server: agent interacts with web pages

The Analogy

If a skill is a recipe, an MCP server is a kitchen appliance. The recipe (skill) tells the chef (agent) what to cook. The appliance (MCP server) gives them the tools to do it.

Security: Not Everything Is Safe to Install

The Reality

  • 13.4% of community skills have critical security issues
  • 36.82% have at least one security flaw
  • Skills can read your files, run commands, and access your system
  • A malicious skill could exfiltrate your code or credentials

The Rules

  • Never install a community skill without reading its SKILL.md first
  • Only use skills from approved sources (official repos, instructor-reviewed)
  • Review what the skill asks the agent to do
  • If it asks for credentials or network access, stop and ask why

The Analogy

Installing a skill is like installing a browser extension. It gets access to your environment. Read what it does before you grant that access.

The Orchestrator Pattern

One central agent coordinates all work. It reads the codebase, decides what to do, and delegates to specialists. It never writes a single file.

Think of it as a project manager: they read requirements, make decisions, and tell each specialist what to build. They do not write the code themselves.

Input

Course plan, week content, pedagogical goals

Decision

Which agent handles the task, what skill to load

Output

Slides, exercises, workshops, professor notes

Multiple specialized agents. Dozens of reusable skills. One orchestrator that never writes code. The same pattern you can use for your projects.
