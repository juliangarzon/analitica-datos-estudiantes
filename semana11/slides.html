Week 11, Data Analytics

Hook

What makes a dashboard useful?

Not more charts. Better questions, better filters, better decisions.

Bad dashboard

  • Too many visuals
  • No clear user question
  • Filters nobody needs
  • Looks impressive, says little

Useful dashboard

  • Starts from a question
  • Highlights a few key metrics
  • Lets the user compare and explore
  • Supports a clear insight

Concept

Two tools, two roles

Plotly

  • Interactive charts
  • Hover, zoom, legend toggles
  • Fast path for visual exploration

Streamlit

  • Turns Python into a web app
  • Adds filters, layout, metrics and tables
  • Lets users explore without editing notebooks
Mental model: Plotly makes the charts interactive. Streamlit makes the whole analysis interactive.

Pattern

The reusable dashboard pattern

load data rename and clean columns create sidebar filters filter the dataframe compute KPIs build charts optionally show the filtered table

If you understand this sequence, you can adapt the same structure to many public datasets.

Pattern

Minimum app requirements

Filters

At least 3, and they must help comparison.

KPIs

At least 3, and they must summarize the current selection.

Charts

At least 3, and each one should answer a different question.

Tips

Common mistakes

Technical

  • Empty multiselect, empty dataframe
  • Dates still stored as strings
  • Too many categories in a single chart

Analytical

  • Charts without a question
  • Metrics without context
  • Titles that say what the chart is, not what it shows
Challenge

Final challenge

Choose another public dataset and reuse the template.

  • Your dataset needs one time variable, one category and one numeric measure.
  • Your app should answer a clear question in less than 30 seconds.
  • Your team should be able to explain why each filter exists.
Wrap-up

What matters from this week

  • Interactive apps are not about showing more, but about exploring better.
  • Plotly handles the interactive chart layer.
  • Streamlit handles the app structure and user flow.
  • A good dashboard pattern can be reused across datasets and projects.
